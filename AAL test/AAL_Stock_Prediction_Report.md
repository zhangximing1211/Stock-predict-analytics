# AAL 股票涨跌预测 — 完整实验报告

## 1. 项目概述

本项目基于 American Airlines (AAL) 的历史交易数据，使用机器学习方法构建**次日涨跌二分类预测模型**。采用滚动窗口回测框架，结合 GridSearchCV 超参数调优，评估模型在真实时间序列场景下的泛化能力。

- **股票代码**：AAL（American Airlines Group Inc.）
- **数据区间**：2013-03-11 至 2018-02-07
- **预测目标**：次日涨跌（二分类：1=涨，0=跌）
- **预测日期**：2018-02-08

---

## 2. 数据说明

### 2.1 数据来源

原始数据文件：`AAL_2013-03-11_to_2018-02-07_all_data.csv`

### 2.2 特征工程

选取 **10 个特征**作为模型输入：

| 特征名 | 说明 |
|--------|------|
| `open` | 开盘价 |
| `high` | 最高价 |
| `low` | 最低价 |
| `close` | 收盘价 |
| `volume` | 成交量 |
| `RSI_14` | 14 日相对强弱指标 |
| `MACD` | 移动平均收敛/发散指标 |
| `BB_width` | 布林带宽度 |
| `volatility_20` | 20 日波动率 |
| `daily_return` | 当日收益率 |

### 2.3 目标变量

```
target = 1  如果次日收益率 > 0（涨）
target = 0  如果次日收益率 ≤ 0（跌）
```

### 2.4 数据预处理

- 日期列转为 `datetime` 格式，按时间升序排列
- 删除缺失值
- 基于 `daily_return` 的 shift(-1) 构造次日收益率，再二值化为目标变量
- 特征标准化：每个滚动窗口内，仅在训练集上 `fit` StandardScaler，再 `transform` 验证集和测试集（防止数据泄漏）

---

## 3. 滚动窗口回测设计

为模拟真实投资场景，采用**前向滚动窗口（Walk-Forward）**验证方式，避免未来信息泄漏。

### 3.1 窗口参数

| 参数 | 设置 |
|------|------|
| 训练窗口 | 36 个月 |
| 验证窗口 | 1 个月 |
| 测试窗口 | 1 天（单日预测） |
| 滚动步长 | 1 天 |

### 3.2 窗口划分逻辑

对于每个测试日 $t$：

$$
\text{训练期} = [t - 37\text{月},\ t - 1\text{月} - 1\text{天}]
$$

$$
\text{验证期} = [t - 1\text{月},\ t - 1\text{天}]
$$

$$
\text{测试期} = t\ \text{（单日）}
$$

每次滚动步长为 1 个交易日，窗口整体向前平移。

### 3.3 窗口统计

- **总窗口数**：462 个
- **首个测试日**：数据起始日 + 37 个月后的第一个交易日
- **最后测试日**：2018-02-07

---

## 4. 模型训练

### 4.1 模型选择

使用两种集成学习模型：

| 模型 | 说明 |
|------|------|
| **Random Forest** | Bagging 集成，sklearn `RandomForestClassifier` |
| **LightGBM** | Boosting 集成，`LGBMClassifier` |

### 4.2 超参数搜索空间

**Random Forest：**

| 参数 | 搜索范围 |
|------|---------|
| `n_estimators` | [100, 200] |
| `max_depth` | [5, 10, None] |
| `min_samples_split` | [2, 5] |

共 $2 \times 3 \times 2 = 12$ 种组合

**LightGBM：**

| 参数 | 搜索范围 |
|------|---------|
| `n_estimators` | [100, 200] |
| `max_depth` | [3, 5, 7] |
| `learning_rate` | [0.01, 0.1] |
| `num_leaves` | [15, 31] |

共 $2 \times 3 \times 2 \times 2 = 24$ 种组合

### 4.3 调参方式

- **方法**：GridSearchCV（网格搜索 + 交叉验证）
- **交叉验证折数**：5 折
- **评估指标**：Accuracy
- **并行**：`n_jobs=-1`（全部 CPU 核心）

### 4.4 训练流程

每个滚动窗口内执行以下步骤：

1. **StandardScaler** 在训练集上 fit，transform 训练/验证/测试集
2. 对 RandomForest 和 LightGBM 分别执行 GridSearchCV
3. 取最优参数模型，在验证集和测试集上预测
4. 保存每个窗口的最优模型（joblib 序列化）

### 4.5 模型保存

- 每个窗口保存 2 个模型：`{ModelName}_window_{i}.joblib`
- 最后一个窗口的模型额外保存为 `{ModelName}_final.joblib`（用于未来预测）
- 最后一个窗口的 Scaler 保存为 `last_scaler.joblib`
- **总计保存**：927 个文件（462 × 2 + 3）

## 5. 模型评估

### 5.1 评估方式

- **验证集评估**：每个窗口的验证集（约 1 个月交易日）计算准确率，取所有窗口的平均值
- **测试集评估**：所有 462 个窗口的单日预测汇总后计算整体指标

### 5.2 评估指标

| 指标 | 说明 |
|------|------|
| Accuracy | 总体预测正确率 |
| Precision | 预测为"涨"中实际为涨的比例 |
| Recall | 实际为"涨"中被正确预测的比例 |
| F1 Score | Precision 和 Recall 的调和平均 |
| 混淆矩阵 | 分类结果的详细分布 |

|------|-----|
| 平均验证准确率 | 0.5053 |
| 测试准确率 | 0.5152（238/462） |
| Precision | 0.4908 |
| Recall | 0.3620 |
| F1 | 0.4167 |

**混淆矩阵：**

|  | 预测跌 | 预测涨 |
|--|--------|--------|
| **实际跌** | 158 | 83 |
| **实际涨** | 141 | 80 |

**分类报告：**

| 类别 | Precision | Recall | F1-Score | Support |
|------|-----------|--------|----------|---------|
| 跌 | 0.53 | 0.66 | 0.59 | 241 |
| 涨 | 0.49 | 0.36 | 0.42 | 221 |
| **Accuracy** | | | **0.52** | **462** |

### 5.4 LightGBM 评估结果

| 指标 | 值 |
|------|-----|
| 平均验证准确率 | 0.5306 |
| 测试准确率 | 0.5238（242/462） |

**混淆矩阵：**

|  | 预测跌 | 预测涨 |

### 5.5 模型对比汇总

| 模型 | 验证准确率 | 测试准确率 | Precision | Recall | F1 |
|------|-----------|-----------|-----------|--------|-----|

> **LightGBM 在所有指标上均优于 RandomForest**，为本实验的最优模型。

## 6. 预测结果

使用最后一个滚动窗口训练的最优模型，基于 2018-02-07 的特征数据，预测 **2018-02-08** 的涨跌方向。

### 6.1 输入特征（2018-02-07）

模型读取 2018-02-07 当天的 10 个特征值，经最后一个窗口的 StandardScaler 标准化后输入模型。

### 6.2 预测输出

| 模型 | 预测方向 | 跌概率 | 涨概率 | 置信度 |
|------|---------|--------|--------|--------|
| RandomForest | **跌 ↓** | 0.5552 | 0.4448 | 0.5552 |

> **两个模型一致预测 2018-02-08 AAL 股价下跌**，LightGBM 置信度更高（58.98%）。

## 8. 结论与讨论

1. **模型表现**：两个模型的测试准确率均在 **52%** 左右，略高于随机猜测（50%），这在股票日频涨跌预测中属于正常水平。金融市场具有高噪声、弱信号的特点，纯技术指标难以实现高精度预测。

2. **LightGBM 更优**：LightGBM 在验证集和测试集的所有指标上均优于 RandomForest，表明 Boosting 方法在捕捉弱信号上有一定优势。

3. **偏向预测"跌"**：两个模型均倾向于预测"跌"（Recall 涨 < Recall 跌），这可能与数据中跌的样本稍多有关（241 跌 vs 221 涨）。

4. **改进方向**：
   - 引入更多特征（如情绪数据、宏观经济指标、行业联动）
   - 尝试深度学习模型（LSTM、Transformer）
   - 调整分类阈值或使用概率校准
   - 增加更长的历史数据
   - 考虑多标签预测（涨幅区间）替代简单二分类
